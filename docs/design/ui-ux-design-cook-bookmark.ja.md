# UI/UXデザイン仕様書 - Cook Bookmark

**作成日**: 2026-02-14
**デザインアプローチ**: モバイルファースト、ウォームミニマル
**技術スタック**: Next.js 16 + Tailwind CSS v4 + shadcn/ui + Lucide React

---

## 1. デザインコンセプト

### コンセプト: 「キッチンコンパニオン」

キッチンでスマートフォンを片手に使えるよう設計された、温かく清潔感のある使いやすいレシピアプリ。可読性、大きなタッチターゲット、レシピへの素早いアクセスを優先します。

### デザイン原則

1. **モバイルファースト**: 全画面を375px幅から設計し、拡大対応
2. **片手操作**: ボトムナビゲーション、届きやすいアクション配置
3. **一覧性**: カードベースのレイアウト、明確な視覚的階層
4. **温かみ**: 食に着想を得たウォームカラーパレット
5. **素早いアクセス**: レシピを見つけて表示するまでの最小タップ数

## 4. 画面設計

### 4.1 ランディングページ（`/`）

- 中央配置、ミニマル
- 単一CTA: Googleログイン
- アプリ説明は日本語
- 温かみのある背景

### 4.2 ダッシュボード / レシピ一覧（`/dashboard`）

- **固定ヘッダー**: アプリ名 + ユーザーアバター
- **検索バー**: ヘッダー下に常時表示
- **カテゴリチップス**: 横スクロール可能
- **レシピカード**: モバイルで2カラムグリッド
- **カード情報**: 画像、タイトル、星評価、お気に入りアイコン、調理時間
- **ボトムナビ**: ホーム（一覧）、追加（新規レシピ）、マイページ

### 4.3 レシピ詳細（`/dashboard/recipes/[id]`）

- **ヒーロー画像**: フル幅でトップに配置
- **メタ情報行**: 評価、時間、人数を一行に
- **セクション分割**: 材料、手順、栄養情報を明確に分離
- **材料リスト**: 分量を右揃えにしたクリーンなリスト
- **手順**: 番号付き、ステップ間に十分な余白
- **元URLリンク**: 元のレシピページへのリンク

### 4.4 レシピ追加（`/dashboard/recipes/new`）

- **ステップ1**: URL入力 + 「AIで取得する」ボタン
- **ローディング**: AI抽出中のスピナーとメッセージ
- **ステップ2**: 抽出データのプレビュー/編集
- **ステップ3**: 保存ボタン
- 保存前にすべてのフィールドを編集可能

### 4.5 レシピ編集（`/dashboard/recipes/[id]/edit`）

- レシピ追加と同じレイアウト、既存データで事前入力
- 画面下部に削除ボタン

## 8. インタラクション

### タッチターゲット
- 全ボタン: 最小 `h-11`（44px）
- カードタップ: カード全体がタップ可能
- お気に入り: アイコンタップで即座にトグル（楽観的更新）
- 評価: 星をタップして設定

### トランジション
- ページ遷移: `transition-all duration-200`
- カードホバー（デスクトップ）: `hover:shadow-md transition-shadow duration-200`
- お気に入りトグル: `transition-colors duration-150`

### ローディング状態
- レシピ一覧: スケルトンカード
- AI抽出: スピナー + 「レシピを解析中...」メッセージ
- 画像読み込み: アスペクト比付きスケルトン

### エラー状態
- AI抽出失敗: リトライオプション付きトースト
- ネットワークエラー: 「接続を確認してください」トースト
- 空状態: 「レシピを追加しましょう」メッセージ

## 9. アクセシビリティ

- `lang="ja"` をhtml要素に設定
- 全画像: `alt`テキスト（レシピ名）
- フォーム入力: `<label>`要素との関連付け
- 星評価: `aria-label="評価: X点"` + キーボード操作可能
- お気に入りボタン: `aria-label="お気に入りに追加/解除"`
- ボトムナビ: `<nav>` + `aria-label="メインナビゲーション"`
- カラーコントラスト: 全テキストで4.5:1以上
- `prefers-reduced-motion`: ユーザー設定を尊重
